By John Doe

Published 2017-02-11

创建GitHub仓库

  1. 注册GtiHub
  2. 新建公共仓库，仓库命名规则为：*.github.io。以我的为例：myblog.github.io

环境准备

安装Git

下载地址：GitHub for windows
安装完成后包括GitHub客户端和GitShell：

  1. 打开GitHub客户端，输入账户名和密码，登录GitHub
  2. 在本地新建GitHub目录（可以是任意位置），将之前创建的远程仓库clone到该目录下（如：E:\MyDocuments\GitHub\myblog.github.io），即本地仓库

PS：无需配置SSH密钥

安装Node.js

下载地址：Node.js for windows
下载直接安装即可

安装Hexo

打开GitShell，输入npm install -g hexo。如果提示找不到npm命令，请首先安装Node.js

部署本地博客

  1. 在GitHub下新建hexo目录
  2. 打开GitShell，定位到hexo目录。输入hexo init，进行hexo初始化。命令结束后，hexo文件夹下会生成一系列的文件。
  3. 将hexo下所有的目录和文件拷贝至本地仓库

至此，本地hexo博客已经搭建好了。本地博客查看方式：

打开http://localhost:4000/，即可看到本地博客。

Hi! Liang! 我的第一篇独立博客！

By John Doe

Published 2017-02-11

搭建初衷

“搭建一个只属于我自己的博客！”

去年就在考虑这个事。期间百度、谷歌了很多教程，但是一般还没准备动手，就已经放弃了。一方面觉得搭建个人博客是一个非常花时间花精力的事情；另外，对于我这个半路出家的半吊子码农来说，未知的东西实在太多，什么GitHub，还有那一堆眼花缭乱的html标签、JS、CSS代码（虽然不知道在具体搭建博客时会不会用到，但还是生畏，这就是对于未知的恐惧吧），实在是看不下去。。。

2017年，对于我来说，注定是不平凡的一年！（后续更新）

既然想干，那就先从我的独立博客开始吧。

从GitHub建仓到买域名再到整个博客初具雏形，花了整整一个白天（也没想象中那么困难、那么费时间），整个过程还是挺有意思的。

之前我也在CSDN、博客园、简书、FarBox上写过博客，但是只在FarBox上坚持了将近半年，其他的现在连帐号都忘了（好个不专一的男人……）。写博客主要是将自己在学习过程中遇到的难题的解决方案记录下来、方便日后查看。之所以坚持要搭建一个独立博客，我觉得有以下几个原因：

  1. 简洁。没有广告。
  2. 装逼。拥有专属自己的域名。即使是helloliangxw.farbox.com，那也不是我的，那只是租的别人的地。一定要是helloliangxw.com，那才是咱家的自留地（郑重声明：本质上还是国家的6666）。
  3. 发扬共享精神。新一代的国家栋梁（我目前还不算，查阅本文的你肯定是23333）就应该有这个精神！我们不断地从开源社区获得那么多的便利与知识，当然也要回馈社区。
  4. 成长记录。任何一个大神都是从菜鸟、小白成长起来的。想想10年后的我，再次打开这些博客，看到这些傻逼的文字、让人失笑的水文，一定会感动到哭的55555555
  5. 决心！既然准备出发，那就从搭建博客开始吧（犹豫了一年多的事情，也该有个了结），算是壮行了！

搭建过程

建站工具，选择Hexo也是无意中的事情（搜索关键词“GitHub”+“博客博客”，几乎所有教程都是用Hexo）。

域名购买，根据教程选择了Godaddy。花了55￥。DNS服务商选的dnspod。

博客托管，GitHub。PS：博客搭建完成后，想换成国内的coding以提高访问速度，试了半个小时左右没有迁移成功。算了，那就忘了它吧。

Markdown，写博客必须用这个。语法很简单。如果您热爱码字但还不知道MarkDown，墙裂建议您学习一下，保证让您眼前一亮：原来码字可以这么优雅！

PS: 有时间将详细的搭建过程写下来！

花了钱，也花了时间，希望自己能够坚持下去！也以此为动力，驱动自己去深入学习一些技术，锻炼自己的总结能力，如果在此过程中能够帮助到其他人，那也算是行善积德了。
